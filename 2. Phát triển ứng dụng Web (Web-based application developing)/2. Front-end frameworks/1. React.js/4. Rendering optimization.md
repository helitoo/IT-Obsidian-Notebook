
# `React.memo`

**Tình huống đặt ra**: Xem xét component function sau:
```jsx
function Parent() {
  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
      
      <Child name="Bảo"/>
    </>
  );
}
```
Nếu `count` thay đổi thì `<Child />` cũng phải re-render theo `Parent` dù nó không dùng đến `count` -> Gây giảm UX và hiệu năng.

**`React.memo()`**: Chỉ render những component nào thực sự cần render.

**VD**:
```jsx
const Child = React.memo(function Child({ name }) {
  console.log("Child render");

  return <h1>{name}</h1>;
});
// Child chỉ bị re-render khi name thay đổi
```

# `React.lazy`

`React.lazy` cho phép lazy load component, chỉ download JS khi component cần render.
- Suspense hiển thị fallback UI trong khi component đang load.
- Giúp code splitting, giảm initial bundle size.

**VD**: `About.jsx`:
```jsx
// About.jsx

function About() {
  return <h1>About Page</h1>;
}

export default About;
```

Sử dụng `React.lazy`:
```jsx
import React, { lazy, Suspense } from "react";

// Chỉ download khi cần
const About = lazy(() => import("./About"));

function App() {
  return (
    <Suspense fallback={<p>Loading...</p>}>
      <About />
    </Suspense>
  );
}
```

# Windowing / Virtualization

**Virtualization** là kỹ thuật chỉ render items hiện trong viewport, không render toàn bộ danh sách hàng nghìn items.
- Thư viện thường dùng: `react-window` (lightweight), `react-virtualized` / `TanStack` (feature-rich).
- Giảm DOM nodes.
- Cần thiết khi render danh sách lớn gây performance issues.
- Đánh đổi là code phức tạp hơn, khó styling các item có chiều cao động, và cần tính toán vị trí scroll chính xác.

**VD**:
```jsx
import { FixedSizeList } from "react-window";

function Row({ index, style }) {
  return (
    <div style={style}>
      User {index}
    </div>
  );
}

function App() {
  return (
    <FixedSizeList
      height={500}
      width={300}
      itemSize={35}
      itemCount={100000}
    >
      {Row}
    </FixedSizeList>
  );
}
```
