
# Function component

Hàm trả về JSX được gọi là **Function component**.

Quy tắc dùng:
- Chỉ có một tham số là props hoặc không có tham số.
- Tên của function component phải luôn viết hoa chữ cái đầu.
- Gọi function như gọi một thẻ HTML.

VD:
Khai báo component với props:
```jsx
function Welcome(props) {
  return <h2>Hello, {props.name}!</h2>;
}
```
Hoặc với phương pháp destruction (phổ biến):
```jsx
function Welcome({ name }) {
  return <h2>Hello, {name}!</h2>;
}
```
Gọi:
```jsx
<Welcome name="Thái" />
```

# Hook

**Hook** là các hàm đặc biệt ở function component giúp bạn triển khai các tính năng của React như:
- State (dữ liệu của component thay đổi)
- Lifecycle.
- Context.
- Ref.
- Side effects.

**Rules of hooks**:
- Chỉ gọi hook ở top-level, không trong vòng lặp, điều kiện hay hàm lồng.
- Chỉ gọi hooks trong function component hoặc custom hook.

**Các loại hook thường dùng**:
- **`useState()`**: Quản lý state.
- **`useEffect()`**: Quản lý các function liên quan đến component.
- **`useMemo()`**: Cache các *biến* trong component để tránh tính toán lại khi component re-render.
- **`useCallback()`**: Cache các *hàm* trong component để tránh tính toán lại khi component re-render.
- **`React.memo()`**: Cache các *component* để tránh tính toán lại khi không cần thiết. Đây không phải 1 hook nhưng vẫn được đưa vào danh sách này vì nó tương tự với `useMemo` và `useCallback`.
- **`useRef()`**: Lưu trữ các biến xuyên qua những lần component re-render.

## useState

`useState` là hook giúp bạn **kiểm soát dữ liệu (state)** của component.

**Cú pháp**:
```jsx
function Component() {
	const [state, setState] = useState(initState);
	// ...
}
```
`initState` là giá trị khởi tạo, chỉ được khởi tạo 1 lần duy nhất kể cả khi `Component` render mấy lần.
Hàm `useState` trả về 1 mảng có 2 phần tử:
1. **`state`**: Giá trị hiện tại của component. Khi `state` thay đổi (ở mức tham chiếu) thì component sẽ được re-render.
2. **`setState`**: Một hàm có chức năng thay đổi `state`.  `setState` không thay đổi giá trị của `state` ngay lập tức mà phải khi hàm này thực hiện xong, component mới được re-render và giá trị mới của `state` được thay đổi.

**Tại sao không thay đổi giá trị của state bằng toán tử gán mà phải thông qua `setState`?**
- State không phải là một biến JS thông thường.
- State là biến đặc biệt được React quản lý. Thông qua `setState`, React mới biết state đã thay đổi để re-render.

**VD1**: Sử dụng `useState` cơ bản.
```jsx
function App() {
	const [counter, setCounter] = useState(1);
	
	const handleIncrease = () => setCounter(counter + 1);
	
	return (
		<div>
			<h1>Current counter: {counter}</h1>
			<button onClick={handleIncrease}>Increase!</button>
		</div>
	);
}
```
- Đầu tiên, giá trị của `state` là `1` (nhờ vào `useState(1)`).
- Mỗi khi `button` được bấm, `handleIncrease` được kích hoạt, dẫn đến kích hoạt `setCounter(counter + 1)` để tăng `counter` lên 1 đơn vị.
Tóm lại, component trên có chức năng tăng `counter` lên 1 đơn vị mỗi lần `button` được click.

**VD2.1**: Vấn đề re-render của `useState`.
```jsx
const handleIncrease = () => {
	// Sử dụng counter = 1
	setCounter(counter + 1); // counter = 2
	
	// Sử dụng counter = 1
	setCounter(counter + 1); // counter = 2
}
// Kết quả, counter = 2
```
 Dù `setCounter` 2 lần nhưng trên thực tế, `state` chỉ tăng lên 1 mỗi lần nhấn `button`.

**VD2.2**: Dùng `setState` với callback:
```jsx
const handleIncrease = () => {
	setCounter(preState => preState + 1);
	setCounter(preState => preState + 1);
}

// counter = 2
// counter = 3
// Kết quả, counter = 3
```
Callback function là hàm không thực hiện ngay, được dùng để làm tham số truyền vào `setCounter`. Callback được thực hiện khi `handleIncrease` thực hiện xong.

**VD3**: Cập nhật các thuộc tính của state là object:
```jsx
const [info, setInfo] = useState({
	name: "A",
	age: 18,
	address: "TP. HCM"
});

const handleUpdate = () => setInfo({
	...info,
	name: "B"
})
```
Đối với object, cần dùng toán tử spread để giữ lại dữ liệu cũ, sau đó mới thêm trường hoặc sửa giá trị.

## useEffect

`useEffect` là hook giúp bạn triển khai các **hàm** xảy ra liên quan đến component mà hook này gắn vào.

**Cú pháp**:
```jsx
function Component() {
	useEffect(callback, [deps]);
	// ...
}

```
`useEffect` sẽ gọi `callback` dựa vào đối số của `[deps]`. Cụ thể:
- Khi **không khai báo** đối số:
	- `callback` sẽ được gọi mỗi khi component render.
	- Giá trị trả về của `callback` (cleanup function) sẽ được gọi ngay trước khi component bị xóa khỏi DOM.
- Khi đối số là **`[]`**, `callback` chỉ được gọi 1 lần khi component mouting (render lần đầu).
- Khi đối số là **`[deps]`**,`callback` sẽ được gọi mỗi khi giá trị của `deps` thay đổi.

**VD**:
```jsx
useEffect(() => {
  // Được gọi khi component được render lần đầu
  
  return () => {
    // Được gọi ngay trước khi component bị xóa khỏi DOM
  };
}, []);
```

## useMemo và useCallback

`useMemo` là hook cache lại các biến JS của component function, tránh khởi tạo lại các biến đó khi component re-render không cần thiết.

Cách thức hoạt động của nó tương tự như `useEffect`.

`useCallback` cũng tương tự `useMemo` nhưng áp dụng cho các hàm.

**VD1**: Khi không `useMemo`:
```jsx
function App() {
  const [count, setCount] = useState(0);

  const expensiveValue = heavyCalculation();
  // Mỗi lần App re-render là expensiveValue phải tính toán lại từ đầu

  // ...
}
```
Dùng `useMemo`:
```jsx
const expensiveValue = useMemo(() => {
  return heavyCalculation();
}, []); // Chỉ tính toán expensiveValue khi component render lần đầu
```

**VD2**:
```jsx
const handleClick = useCallback(() => {
  // ...
}, []);
```

## React.memo

**Tình huống đặt ra**: Xem xét component function sau:
```jsx
function Parent() {
  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
      
      <Child name="Bảo"/>
    </>
  );
}
```
Nếu `count` thay đổi thì `<Child />` cũng phải re-render theo `Parent` dù nó không dùng đến `count` -> Gây giảm UX và hiệu năng.

**`React.memo()`**: Chỉ render những component nào thực sự cần render.

**VD**:
```jsx
const Child = React.memo(function Child({ name }) {
  console.log("Child render");

  return <h1>{name}</h1>;
});
// Child chỉ bị re-render khi name thay đổi
```

## useRef

`useRef` dùng để **lưu trữ một giá trị giữa các lần re-render** của component mà việc thay đổi giá trị đó không làm component re-render lại.

**Cú pháp**:
```jsx
const ref = useRef(initialValue);
```
`useRef` trả về 1 object có thuộc tính `current: initialValue`.

**VD1**: Lưu giá trị giữa các lần render:
```jsx
function App() {
  const countRef = useRef(0);

  const handleClick = () => {
    countRef.current++;
  };

  return <button onClick={handleClick}>Click</button>;
}
```

**VD2**: Lưu vết giá trị lần render trước đó:
```jsx
function App() {
  const [count, setCount] = useState(0);
  const prevCount = useRef();

  useEffect(() => {
    prevCount.current = count;
  }, [count]);

  return (
    <>
      <p>Current: {count}</p>
      <p>Previous: {prevCount.current}</p>
    </>
  );
}
```

**VD2**: Thao tác với DOM thông qua `useRef()`:
```jsx
import { useRef } from "react";

function App() {
  const inputRef = useRef();

  const handleFocus = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} />
      <button onClick={handleFocus}>
        Focus
      </button>
    </>
  );
}
```
`inputRef` được gắn vào thẻ `<input>`. Khi `handleFocus()` được gọi, `<input>` sẽ được `.focus()`.

## Custom hook

**Custom Hook** là một hàm JavaScript thông thường có tên bắt đầu bằng `use`, bên trong có thể sử dụng các Hook khác của React (`useState`, `useEffect`, `useRef`, ...), dùng để **tái sử dụng logic** liên quan đến state hoặc lifecycle giữa nhiều component.

**So sánh utility functions với custom hooks ?**
- Dùng **utility function (pure function)** khi: chỉ xử lý data transformation, validation, formatting - không cần React context,... những tác vụ này chung chung, có thể gọi ở bất kỳ đâu.
- **Custom hook** chỉ gọi được từ components.

**VD**:
```js
function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increase = () => setCount(c => c + 1);
  const decrease = () => setCount(c => c - 1);

  return { count, increase, decrease };
}
```
Sử dụng:
```jsx
function App() {
  const { count, increase } = useCounter();

  return (
    <>
      <h1>{count}</h1>
      <button onClick={increase}>+</button>
    </>
  );
}
```
