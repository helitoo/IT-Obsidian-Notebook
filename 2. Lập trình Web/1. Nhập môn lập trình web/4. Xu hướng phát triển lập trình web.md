# Server side rendering (SSR)

**SSR** là server sẽ trả về HTML, browser chỉ cần hiển thị HTML đó.

**Ưu điểm**:
- Tốt cho SEO (Search Engine Optimization), nội dung trang web có thể dễ dàng đọc hiểu bởi browser và đưa lên gợi ý hàng đầu khi tìm kiếm web.
- Load trang lần đầu tiên sẽ rất nhanh.
- Sẽ rất tuyệt vời đối với các static page.

**Nhược điểm**:
- Server vừa đảm nhận logic, vừa đảm nhận HTML có thể gây áp lực lớn cho server, tăng độ trễ khi truy cập web, giảm UX.
- Trang web phải xử lý lại hoàn toàn và load lại từ đầu nếu chỉ có một thay đổi nhỏ trong nội dung.

Đây là lý do CSR ra đời.

# Serverless & Backend-as-a-Service

Cùng với sự phát triển của CSR, server chỉ đóng vai trò là nơi trung gian giữa database và client là chính. Các dịch vụ serverless và Backend-as-a-Service ra đời.

| Tiêu chí                         | Serverless                                                                                                                 | Backend-as-a-Service (BaaS)                                                                      |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **Định nghĩa**                   | Mô hình **chạy mã không cần quản lý server**, bạn chỉ triển khai _function_ hoặc _service_ và hệ thống tự lo phần còn lại. | Cung cấp **hệ thống backend sẵn có** để frontend kết nối trực tiếp mà không cần tự viết backend. |
| **Ví dụ**                        | - AWS Lambda.<br>- Cloudflare Workers.<br>- Google Cloud Functions.<br>- Vercel, Netlify,...                               | - Firebase, Supabase.<br>- Appwrite.<br>- AWS Amplify.<br>- ...                                  |
| **Bạn phải code backend không?** | Có.                                                                                                                        | Không hoặc rất ít, các dịch vụ được cung cấp sẵn qua API.                                        |
| **Phù hợp với**                  | Các dự án cần xử lý nghiệp vụ phức tạp, workflow tùy chỉnh.                                                                | Các app nhỏ, MVP, startup cần làm nhanh.                                                         |

>[!note]
>**Hosting**: Nơi chứa web.
>**Serverless**: Chạy backend/web mà không quản lý server thủ công.
>**BaaS**: Backend trọn gói (Auth + Database + Hosting + Function), sử dụng qua API.

# Hybrid system

**Hybrid system** là các trang web SSR nhưng bổ sung các thế mạnh của CSR. Có thể coi hybrid là dạng *fullstack developing*.
