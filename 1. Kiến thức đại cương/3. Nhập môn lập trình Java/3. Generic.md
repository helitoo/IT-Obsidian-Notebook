Generic là cách viết tham số hóa kiểu dữ liệu.

**Cú pháp**: Định nghĩa trực tiếp generic trong định nghĩa biến, hàm, class,...
- `<A>`: Kiểu dữ liệu đơn.
- `<A, B>`: Kiểu tuple (record).
- `<A, B, C,...>`
Với `A`, `B` là các tham số kiểu dữ liệu.

**Bounded type**: Bạn có thể chỉ thị các kiểu nhận được của `T` bằng từ khóa `extends` và `super`. VD:
- `<T extends Number>`: `T` chỉ chấp nhận các kiểu con của `Number` (`Byte`, `Short`, ...).
- `<T extends Character>`: `T` chỉ chấp nhận kiểu `char`.`
- `<T extends Boolean>`: `T` chỉ chấp nhận kiểu `boolean`.
- `<T extends List>`: `T` chỉ chấp nhận kiểu interface của `List` collection.
- ...
- `<T super B>`: `T` chỉ chấp nhận kiểu `B` hoặc cha của `B`. Trong đó, `Object` là kiểu cha lớn nhất.

**Wildcard**: Là ký hiệu `?` đại diện cho một kiểu không xác định, kiểu này được bổ sung sau.
VD:
```java
Collection<?> c = new ArrayList<String>; // ? đã được xác định bằng new ArrayList<String>
```

**Generic class / Generic method**: Là các kiểu dữ liệu mà class/method có thể xử lý. Cú pháp: `class C<T1, T2, T3, ...>`,... (tương tự với method).

VD: Generic class:
```java
class Box<T> {
	T value;
	
	void set(T value) {  
		this.value = value;  
	}  
	
	T get() {  
		return value;  
	}  
}
```
Khi gọi `Box<String> stringBox = new Box<>();` sẽ tạo ra một đối tượng có kiểu `T` là `String`.

VD: Generic method:
```java
public static <T> void printArray(T[] array) {  
	for (T item : array) {  
		System.out.println(item);  
	}  
}
```
