Collection là các cấu trúc dữ liệu nâng cao ngoài `String` và array, hỗ trợ sẵn trong Java.

# Duyệt Collection

## Duyệt bằng `for`

Cả `List` và `Set` đều có thể được duyệt bằng [[2.1. Cú pháp cơ bản#Dùng for-each|for-each]]. `Map` được duyệt gián tiếp thông qua `.keySet()`, `.values()`, hay `.entrySet()`.

`List` có thể được duyệt bằng [[2.1. Cú pháp cơ bản#Dùng for|for]] thông thường vì nó có phương thức `.get()` để lấy phần tử.

**VD**: Duyệt `Iterable`:
```java
for (String s : itrb) {
    System.out.println(s);
}

// Chỉ áp dụng với list
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

**VD**: Duyệt `Map`:
```java
// Duyệt qua key
for (String key : map.keySet()) {
    System.out.println(key);
    System.out.println(map.get(key));
}

// Duyệt cặp key - value
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " : " + entry.getValue());
}
```

## Duyệt bằng `Iterator`

`Iterator` là một lớp của `java.util` dùng để duyệt các collection.

**Mô hình**:
```java
Iterator<String> it = collection.iterator();

while(it.hasNext()) {
	System.out.println(it.next());
}
```
Trong đó:
- `.iterator()`: Là phương thức khởi tạo đối tượng `Iterator` của các collection.
- `.hasNext()`: Trả về `true` nếu còn phần tử liền tiếp. Tương tự `.hasPrevious()`.
- `.next()`: Trả về phần tử hiện tại và dịch chuyển đến phần tử liền kề tiếp theo. Tương tự `.previous()`.

Các phương thức khác:
- `.remove()`: Xóa phần tử hiện tại.

**VD**: Duyệt `Iterable`:
```java
Iterator<String> it = itrb.iterator();

while (it.hasNext()) {
    System.out.println(it.next());
}
```

**VD**: Duyệt `Map`:
```java
// Duyệt qua key (không lấy được value)
Iterator<String> it = map.keySet().iterator();

while (it.hasNext()) {
    System.out.println(it.next());
}

// Duyệt cặp key - value
Iterator<Map.Entry<String, Integer>> it = map.entrySet().iterator();

while (it.hasNext()) {
    Map.Entry<String, Integer> entry = it.next();
    System.out.println(entry.getKey() + " : " + entry.getValue());
}
```

# Một số collection

## Interface `List`

`List` là mảng động chứa giá trị bất kỳ. Có 2 class implement đến là `ArrayList` và `LinkedList`.
- : Khi thêm / xóa phần tử mới -> tạo mảng mới bằng cách copy mảng cũ + hiệu chỉnh phần tử, tốn $O(n)$.
- `LinkedList`: Là một danh sách liên kết đôi, có thể trực tiếp thêm/xóa phần tử với $O(1)$, nhưng chi phí duyệt là $O(n)$.

## Interface `Set`

`Set` là cấu trúc dữ liệu tập hợp, *các phần tử không được trùng nhau*, không thể truy xuất ngẫu nhiên (nhưng vẫn có thể for-each).

Có 3 class implement:
- `HashSet`: Các key thêm vào được sắp xếp *ngẫu nhiên*.
- `LinkedHashSet`: Các key thêm vào được sắp xếp *theo thứ tự được thêm vào*.
- `TreeSet`: Các key thêm vào được sắp xếp theo thứ tự *tăng dần* -> Tốc độ chậm hơn.

## `Map`

`Map` là cấu trúc dữ liệu mà mỗi phần tử có dạng key - value. Key không thể trùng nhau.

Có 3 class implement:
- `HashMap`: Các key được sắp xếp *ngẫu nhiên*. Cho phép 1 key là `null` và các value là `null`.
- `LinkedHashMap`: Các key được sắp xếp *theo thứ tự được thêm vào*. Cho phép 1 key là `null`.
- `TreeMap`: Các key sắp xếp theo thứ tự *tăng dần* -> Tốc độ chậm hơn. Không cho phép key / value là `null`.

# `Collections`

`Collections` là một lớp của `java.util`, chứa các phương thức tĩnh để xử lý collection như:
- `Collections.sort(urClt);`: Sắp xếp `urClt` theo thứ tự tăng dần.
- `Collections.sort(urClt, Collections.reverseOrder());`: Sắp xếp `urClt` theo thứ tự giảm dần.
- `Collections.reverse(urClt);`.
- `Collections.max(urClt)`, `Collections.min(urClt)`.
- `Collections.fill(urClt, val);`: Thay thế tất cả phần tử của `urClt` thành `val`.
- `Collections.binarySearch(urClt, val)`: Trả về index của giá trị `val` trong `urClt` (đã được sắp xếp tăng dần sẵn). Nếu không tìm thấy, nó sẽ trả về index liền trước vị trí đáng nhẽ phải xuất hiện `val`.
