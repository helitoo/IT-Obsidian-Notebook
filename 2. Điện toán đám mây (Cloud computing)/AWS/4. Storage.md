# Block storage VS Object storage

**Block storage**: Lưu dữ liệu ở từng block cụ thể, khi cần chỉnh sửa thì chỉ cần sửa những block lưu dữ liệu đó, có tính linh hoạt cao và phổ biến.

**Object storage**: Là lưu trữ toàn bộ dữ liệu như là một object, khi cần chỉnh sửa một điểm dữ liệu nào đó thì cần cập nhật toàn bộ object chứa nó.

# Amazon Elastic Block Store (Amazon EBS)

Cung cấp dịch vụ block storage cho các EC2 instance, được tích hợp vào trong dịch vụ EC2.

Đơn vị lưu trữ là **volume**. Volume là 1 thực thể được tạo ra độc lập và chỉ có thể được sử dụng khi đính vào EC2 instance (Attaching).

- **Encyption**: Mã hóa dữ liệu, không có chi phí phát sinh.
- **Elasticity**: Có thể thay đổi không gian lưu trữ và phân loại storage.

# Amazon Simple Storage Service (Amazon S3)

Đơn vị lưu trữ là bucket.

**Truy cập vào bucket**:
- **Bucket path-style URL endpoint**: `https://s3.region-name.amazonaws.com/bucket-name`.
- **Bucket virtual hosted-style URL endpoint** (phổ biến hơn): `https://bucket-name.s3.region-name.amazonaws.com`.

# Amazon Elastic File System (Amazon EFS)

Amazon EFS là dịch vụ lưu trữ các file hệ thống (file systems) của các EC2 instance.

Dữ liệu của EFS có thể được chia ra lưu trữ trên nhiều server khác nhau. Các server này có phương thức truy cập như nhau, và quy trình đồng bộ dữ liệu do dịch vụ này đảm nhận.
