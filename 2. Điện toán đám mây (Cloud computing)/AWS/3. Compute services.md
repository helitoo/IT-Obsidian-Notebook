# Amazon EC2 (Amazon Elastic Compute Cloud)

### Tổng quan

- Cung cấp máy ảo (**EC2 instances**) trên cloud.
- Cho phép người dùng toàn quyền điều khiển đến tận OS (*thường là Windows, Ubuntu, Amazon Linux,... với mỗi chính sách khác nhau*), kiểm soát lưu lượng vào ra của mỗi instance.
- Có thể triển khai instance với nhiều kích thước khác nhau ở nhiều Availability khác nhau thông qua **Amazon Machine Images (AMIs)**.

### 9 bước xây dựng instance

1. **AMI**:
	- AMI Là một template để tạo ra các instance.
	- Bao gồm OS và nhiều tùy chọn trước khi khởi tạo instance.
	
2. **Instance type**:
	- Quy định các cấu hình của instance (*RAM, CPU, Storage, Network performance,...*).
	- Phân loại chức năng của instance (*General purpose, compute optimized, memory optimized, accelerated computing,...*).
	- Tên instance: Gồm có Family name, generation number, size.
	- VD:
	![[amazon-ec2-instance-creating.png]]
	
3. **Network settings**:
	- Xác định VPC, subnet (nếu có) mà instance thuộc về,
	- Nên chọn 1 public IP address để gán lên instance để nó có thể kết nối với instance.
	
4. **IAM role**: Kiểm soát quyền truy cập của những AWS services khác đối với instance.
	
5. **User data script**: Là những bash script được khởi chạy dưới quyền root khi instance được khởi tạo và chạy thành công.
	
6. **Storage options**:
	- Cấu hình nơi lưu trữ dữ liệu của instance (**root volume, additonal storage volumes**).
	- Volume là đơn vị lưu trữ dữ liệu của instance, gồm có size (*GB*), type (*SSD, HDD*), encryption,...
	- **Amazon EBS (Amazon Elastic Block Store)**: Lưu trữ các volume như là các block, dữ liệu được lưu trữ ngay cả khi instance restart.
	- **Amazon EC2 Instance Store**: Lưu trữ các volume trên ổ đĩa cứng trên thiết bị người dùng (host computer). Không gian lưu trữ này có tính *Ephemeral* và sẽ bị mất đi khi instance stop.
	
7. **Tags**: Chứa các metadata của instance.
	
8. **Security groups**: Cấu hình lưu lượng vào ra instance.
	
9. **Key pair**:
	- Là cặp mã khóa dùng để xác thực xem một người dùng có thể truy cập vào instance không.
	- **Public key**: Có thể do nhiều người sỡ hữu.
	- **Private key**: Chỉ do chủ sỡ hữu instance sỡ hữu và chỉ có thể tải xuống duy nhất một lần khi khởi tạo instance.
	- **Password**: Là mật khẩu đăng nhập vào instance, chỉ có thể có được nhờ vào private key. Password có thể là administrator (Windows AMI) hay SSH (Linux AMI).

# AWS Lambda

Là dịch vụ điện toán ẩn server (serverless). Trong đó người dùng chỉ cần tập trung vào những vấn đề logic nghiệp vụ.
- Hỗ trợ nhiều ngôn ngữ lập trình khác nhau.
- Việc quản lý do AWS đảm nhận.
- Có thể điều phối quản lý nhiều function đồng thời, theo lịch.

# Container services

Là những dịch vụ cung cấp máy ảo (**EC2 instances**) nhưng ở trong Docker container.
- **Amazon ECS (Amazon Elastic Container Serivce)**: Hỗ trợ Docker.
- **Amazon RKS (Amazon Elastic Kubernetes Service)**: Hỗ trợ Kubernetes, trong đó AWS có vai trò quản lý, cấu hình Kubernetes.
- **Amazon ECR (Amazon Elastic Container Registry)**: Cũng tương tự như Docker Hub.
