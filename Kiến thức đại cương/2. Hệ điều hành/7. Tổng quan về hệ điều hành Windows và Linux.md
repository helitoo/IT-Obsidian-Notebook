
|                                     | Windows                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | Linux                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

| **Dựa trên**                        | New technology (NT) dùng Win32 API.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       | UNIX.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Đặc điểm**                        | - Bảo mật nhiều lớp.<br><br>- Đáng tin cậy.<br><br>- Dễ mở rộng với kiến trúc phân tầng.<br><br>- Hiệu năng cao.<br><br>- Tương thích cao: Có thể chạy trên nhiều chuẩn khác nhau.<br><br>- Di động: Chạy được trên nhiều nền tảng kiến trúc phần cứng khác nhau.                                                                                                                                                                                                                                                                                                                                                         | - Mã nguồn mở.<br><br>- Có nhiều phiên bản do cộng đồng tạo ra (**Bản phân phối - Distribution**). Mỗi bảng phân phối gồm *mã nguồn* hệ thống Linux cơ bản, hệ thống cài đặt, các công cụ quản lý và UNIX.                                                                                                                                                                                                                                                                                         |
| **Kiến trúc hệ thống**              | Modules:<br><br>- **Nhân (kernel)**: Sử dụng ngôn ngữ C là chính. Có nhiệm vụ *định thời, đồng bộ, xử lý ngắt, chuyển đổi giữa user mode và kernel mode*.<br><br>- **Hyper-V hypervisor**: Ảo hóa.<br><br>- **HAL**: Cung cấp giao diện sử dụng phần cứng để phục vụ định thời và các tác vụ điều hành.<br><br>- **Secure kernel**: Hoạt động như kernel mode cho các ứng dụng trustlet (*đạt chuẩn bảo mật của Windows*) và truy xuất đến các khóa bí mật của phần cứng, TPM,...<br><br>- **Plug-and-Play (PnP)**: Tự động nhận diện và cấu hình các thiết bị phần cứng.<br><br>- **Registry**: Chứa thông tin cấu hình. | - **Nhân (Kernel)**: Duy trì các hoạt động ở mức trừu tượng của OS như *virtual memory, process*,... Kernel có quyền truy cập đến mọi resources.<br><br>- **Kernel modules**: Bổ sung tính năng cho kernel mà không cần biên dịch toàn bộ kernel ngay từ đầu.<br><br>- **Thư viện hệ thống (System libraries)**: Định nghĩa chuẩn các hàm có thể tương tác với kernel, có quyền ít hơn kernel.<br><br>- **Công cụ hệ thống (System utilitites)**: Các chương trình thực hiện các chức năng cụ thể. |
| **Quản lý process**                 | Thread có **8 trạng thái** và **2 chế độ thực thi** (*user-mode thread, kernel-mode thread*). Mỗi chế độ có 1 stack riêng để quản lý.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | Có 2 thao tác:<br>- `fork()`: Tạo process.<br>- `exec()`: Chạy chương trình.<br><br>1 process chứa tất cả thông tin mà OS phải lưu trữ để quản lý, gồm có mã định danh, môi trường và ngữ cảnh.                                                                                                                                                                                                                                                                                                    |
| **Giao tiếp giữa các process**      | **Local producedure call facility**: Dùng mô hình client - server.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | Các process giao tiếp với nhau và với kernel qua **signal**. Cả process và kernel đều có thể tạo ra signal, nhưng kernel chủ yếu dùng các trạng thái định thời và hàng đợi.                                                                                                                                                                                                                                                                                                                        |
| **Chuyển dữ liệu giữa các process** | --                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | Có 3 cách:<br>- **Pipe**: Process con kế thừa process cha.<br>- **Network**: UNIX dùng các công cụ mạng để gửi dữ liệu đến các process.<br>- **Shared memory**: Bộ nhớ dùng chung.                                                                                                                                                                                                                                                                                                                 |
| **Định thời**                       | Định thời **32 mức độ ưu tiên**, chia thành 2 lớp:<br>- **Variable**: Ưu tiên từ 0 đến 15 và *có thể thay đổi*.<br>- **Real-time**: Ưu tiên từ 16 đến 31 và *không thể thay đổi*.<br><br>OS đặc biệt ưu tiên các process **interactive** (*sử dụng chuột, cửa sổ,..*).<br><br>-> Hiệu quả cao, thời gian đáp ứng nhanh.                                                                                                                                                                                                                                                                                                   | - **Phiên bản 2.5**: Định thời *ưu tiên* và *trưng dụng* ($O(1)$).<br><br>- **Phiên bản 2.6**: *Completely Fair Scheduler (CFS)*.                                                                                                                                                                                                                                                                                                                                                                  |
| **Quản lý bộ nhớ**                  | - Dynamic paging.<br>- Mỗi trang có **6 trạng thái**.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | - **Bộ nhớ vật lý**: Được chia thành **6 vùng nhớ**, mỗi vùng được cấp trang riêng (*buddy heap*).<br><br>- **Bộ nhớ ảo**:<br>  + Next-fit, demand-paging.<br>  + Chia thành 2 view là **logical view** (cách bố trí không gian địa chỉ) và **physical view** (quản lý bởi page table và routines).                                                                                                                                                                                                |
